üß™ Mengapa Virtual Environment Penting?

terminal ‚Äî bash
$
python -m pip install tensorflow==2.10
$
python -m pip install tensorflow==2.15
ERROR: Cannot install tensorflow==2.10 and tensorflow==2.15 because these package versions have conflicting dependencies.

Tanpa virtual environment, Anda akan mengalami konflik dependency seperti di atas. Dengan virtual environment:

  • üîí Isolasi: Package proyek A (TensorFlow 2.10) tidak bentrok dengan proyek B (TensorFlow 2.15)
  • üìå Konsistensi: Tim bisa mereplikasi environment yang sama persis
  • üßπ Kebersihan: Sistem Python global tetap bersih
  • üîÑ Fleksibilitas: Beralih antar proyek dengan mudah

Langkah 1: Install Anaconda

  1. Download installer dari situs resmi Anaconda
  2. Jalankan installer dan ikuti petunjuk
  3. Pilih "Just Me" (direkomendasikan)
  4. Centang "Add Anaconda to PATH" (opsional tapi memudahkan)
  5. Verifikasi instalasi:
Anaconda Prompt
(base) $
conda --version
conda 23.11.0

Langkah 2: Buat Environment Conda

Buat environment untuk proyek data science:

conda create -n data-science python=3.10

Aktifkan environment:

conda activate data-science

üí° Coba Sendiri!

Buat environment baru dengan nama berbeda:

conda create -n my-project python=3.9
conda activate my-project

Langkah 3: Install Package dengan Conda

Package dasar untuk data science:

conda install numpy pandas matplotlib jupyter scikit-learn
Tip: Conda lebih baik untuk package sains yang kompleks seperti TensorFlow atau PyTorch karena sudah dikompilasi sebelumnya.

Langkah 4: Integrasikan UV

Dalam environment Conda, install UV untuk manajemen package yang lebih cepat:

pip install uv

Sekarang Anda bisa menggunakan perintah UV:

uv pip install seaborn plotly

üí° Best Practices

Yang Harus Dilakukan

  • Buat environment terpisah untuk setiap proyek
  • Gunakan Conda untuk package besar (TensorFlow, PyTorch)
  • Gunakan UV untuk kecepatan instalasi package biasa
  • Simpan dependensi di environment.yml dan requirements.txt
  • Tambahkan .venv ke .gitignore

Yang Harus Dihindari

  • Jangan install package global tanpa environment
  • Hindari mencampur Conda dan pip/UV sembarangan
  • Jangan abaikan versi Python saat buat environment
  • Jangan lupa mengekspor dependensi
